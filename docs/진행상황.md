# 현재까지 진행상황

작성일: 2026-06-06
기준 버전: design.md v0.4

이 문서는 지금까지 구현된 내용과 다음 작업을 한눈에 보기 위한 요약이다.
세부 설계 원칙은 [design.md](design.md)를 참고한다.

---

## 한 줄 요약

플레이어가 유닛 명령창에서 이동·정찰·보급 복귀를 지시하고, 유닛이 지도를 따라 이동·정찰하며, 통신망을 통해 (지연될 수 있는) 보고를 받는 핵심 작전 루프가 동작한다.

---

## 구현된 기능

### 1. 정찰 명령 시스템
- 지도에서 구역을 선택하고 정찰병에게 인접 구역 정찰을 명령한다.
- 명령 가능 범위는 유닛의 현재 구역과 인접 구역으로 제한된다(넓은 시야를 주지 않음).
- 명령 → 이동 → 정찰 → 보고 → 다음 명령의 루프가 완성됨.

### 2. 정찰 시스템 재설계
- 기존의 "진행도 게이지(0→100%)가 곧 정보 품질" 모델을 폐기.
- 새 모델: **목표 구역 도착 → 초기 정찰 시간(지형·레벨 기반) → 첫 보고 → 주기적 갱신 보고**.
- 정보 품질(Class)은 진행도가 아니라 **정찰병 레벨**로 결정 (Lv1=D … Lv4+=A).
- 적 규모 추정 정밀도도 레벨에 따라 달라짐 (저레벨은 넓은 범위, 고레벨은 정확).
- 보고 주기도 레벨이 높을수록 짧아짐.

### 3. 유닛 가시성 (지도 토큰 + 유닛 패널)
- 작은 글씨 대신 **NATO식 군용 카운터**로 유닛 표시 (보병=X, 정찰=사선, 포병=점).
- 토큰에 레벨 배지, 상태 경고, 유닛 이름, 실시간 활동 텍스트(이동 중 / 초기 정찰 중 N시간 / 정찰 중·다음 보고 N시간).
- 우측 패널에 **구역 상세 / 유닛** 탭. 유닛 탭은 전체 유닛 목록(종류·위치·활동·HP/식량).
- 토큰이나 유닛 목록에서 선택하면 지도에서 강조 + 구역 동기화.

### 4. 인접 그래프 기반 이동
- 이동 경로를 데이터 배열 순서가 아니라 **구역 인접 관계(neighbors) BFS 최단경로**로 계산.
- 이전엔 인접 구역으로 가도 엉뚱한 곳을 경유하던 버그를 수정. 실제 지도 지형을 따라 이동.

### 5. 통신 시스템 (릴레이 체인 + 보고 버퍼링)
- **연결 판정**: 앵커(HQ·야전사령부)에서 통신범위(2홉) 이내 유닛 연결 → 그 유닛이 다시 송신원이 되어 사슬처럼 확장. 유닛을 중계용으로 배치하면 통신망이 늘어난다.
- **두절 시**: 기존 명령은 계속 수행, 신규 명령 차단, 정찰 보고와 적 정보가 유닛에 **버퍼링**되어 HQ는 보지도 알지도 못함("보는 것 ≠ 아는 것").
- **재연결 시**: 밀린 보고 일괄 전달(지연 표시), 그제서야 적 정보가 지도에 반영, "통신 복구" 보고 발생.
- 표시: 두절 유닛은 "통신두절" 경고, 명령 버튼 대신 "명령 불가" 안내.

### 6. 정리/일관성 작업
- **네이밍 통일**: 내부 그리드 id(A1~D5)는 숨기고, UI에는 항상 구역 코드(Valley A, Plain D…)로 표시.
- **작전 시간 체계**: 내부 시간은 작전 분 단위, 실시간 1초는 작전 15분으로 진행. 클록은 `D1 06:00` 형식, UI는 일/시간/분으로 표시.
- GitHub Pages 캐시 대응: ES 모듈 import에 버전 파라미터(`?v=N`) 부여.

### 7. 보급 시스템
- 보급 거점은 통신 앵커인 **Forest D(HQ)**와 **Plain M(야전사령부)**에서 파생된다.
- 식량은 정수 포인트가 아니라 기준 소모율 기준 작전 가능 시간이며, UI에는 `3일`, `1일 12시간`처럼 표시한다.
- 식량은 대기 중이 아니라 이동·정찰·복귀·교전 같은 작전 행동 중 감소한다.
- 식량 소진 시 가장 가까운 보급 거점으로 자동 복귀한다(BFS 거리 기준).
- 보급 거점 도착 시 식량 회복 + 보급 완료 보고가 생성된다.

### 8. 유닛 명령창
- 지도 위 유닛 토큰 또는 유닛 목록을 클릭하면 유닛 명령창이 열린다.
- 명령창에서 위치·현재 명령·목표·통신·HP/식량을 확인한다.
- 인접 구역 이동, 정찰병 인접 구역 정찰, 가장 가까운 보급 거점 복귀를 명령할 수 있다.
- 정찰 명령은 목표 구역별 도착 시간, 첫 보고 시간, 보고 주기, 복귀 시간, 식량 여유를 보여주고 현장 체류 시간(4시간 / 8시간 / 1일)을 선택한다.
- 계획 체류 시간이 끝난 정찰병은 자동으로 가장 가까운 보급 거점에 복귀한다.
- 구역 상세 패널 아래에 붙어 있던 정찰 명령 버튼은 제거했다.

## 실험 시나리오 (현재 기본 셋업)

- 정찰병 3기(Alpha Lv2 / Bravo Lv2 / Charlie Lv1)가 **Forest D**에서 시작.
- 통신 앵커: **Forest D(HQ)**, **Plain M(야전사령부)**.
- 목표: 북쪽 **Valley A** 방향으로 정찰.
- 통신범위 2홉 기준, 북부(Valley/Ridge)는 두절 구역. 그냥 가면 Ridge A에서 통신이 끊기고,
  **Plain C에 릴레이 유닛을 두면** 끝까지 연결이 유지된다(릴레이 배치 학습용 구도).

## 코드 구조 메모

- `data/map.js` — 구역 데이터(id·code·neighbors·polygon), `getSectorById` / `codeForSector`.
- `game/simulation.js` — 코어: 시간 진행, 이동(BFS), 정찰, 통신 재계산/버퍼링, 보급, 보고.
- `game/unit.js` / `game/sector.js` / `game/report.js` — 모델.
- `ui/map.js` — SVG 지도, 유닛 토큰, 앵커 마커.
- `ui/details.js` / `ui/roster.js` / `ui/operations.js` — 패널.
- `ui/unit-display.js` — 토큰/패널 공용 표시 헬퍼(심볼·활동 텍스트·톤).
- `main.js` — 앱 조립, 게임 루프, 탭/선택 상태, 유닛 명령창.
